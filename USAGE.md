## 1. 系统简介

个人案件管理系统是一款完全本地化运行的案件管理工具，专为律师和法务人员设计。

### 核心特点

- **完全离线** — 所有数据存储于本地 SQLite 数据库，无需联网，无需注册
- **隐私优先** — 案件数据不上传任何服务器，杜绝信息泄露风险
- **一站式管理** — 覆盖案件、客户、传票、文档、时间线、工作记录、待办、模板等全流程
- **轻量敏捷** — 基于 Python Flask + SQLite，安装简单，资源占用低

### 适用场景

- 诉讼律师跟踪案件全生命周期（立案→一审→二审→执行）
- 法务部门管理企业案件与外部律所协作
- 个人律师管理多个案件的工作进度与工作记录

## 2. 安装与启动

### 环境要求

| 依赖 | 最低版本 |
|------|----------|
| Python | 3.9+ |
| pip | 21.0+ |

### macOS

```bash
# 克隆或解压项目后进入目录
cd law-case-manager

# 运行安装脚本（自动安装 Python 依赖并初始化数据库）
bash install_mac.sh

# 启动系统
bash start.sh
```

### Windows

```powershell
# 右键「Windows 一键打包.bat」运行（安装 + 打包为 exe）
# 或手动安装依赖后启动：
pip install -r requirements.txt
python app.py
```

### 手动安装（通用）

```bash
pip install -r requirements.txt
python app.py
```

启动后在浏览器访问：**http://127.0.0.1:5066**

> 💡 首次启动会自动初始化数据库并填充示例案件数据。如需重新填充，先删除 `data/cases.db`，再运行 `python seed.py`。

## 5. 案件管理

### 创建案件

1. 点击左侧栏「案件管理」
2. 点击「新建案件」按钮
3. 填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 案号 | 法院立案编号，如 (2026)苏0211民初字第1234号 | 是 |
| 案件名称 | 案件中英文名称均可 | 是 |
| 案件类型 | 从下拉列表选择（民事诉讼、知识产权、合同纠纷等） | 是 |
| 案件状态 | 进行中 / 已结案 / 暂停 | 否 |
| 受理法院 | 承办法院全称 | 否 |
| 法官 | 承办法官姓名 | 否 |
| 客户名称 | 委托人姓名或企业名称 | 否 |
| 对方当事人 | 被告/被申请人 | 否 |
| 诉求金额 | 标的金额（元） | 否 |
| 律师费 | 收费金额（元） | 否 |
| 收费状态 | 未收费 / 部分收费 / 已收费 | 否 |
| 立案日期 | 案件立案日期 | 否 |
| 案件描述 | 案情概述 | 否 |

### 搜索与筛选

- **搜索栏**：支持案号、案件名称、客户名称模糊搜索
- **状态筛选**：全部 / 进行中 / 已结案 / 暂停
- **排序**：默认按更新时间倒序排列

### 编辑与删除

- 点击案件行进入详情页，可编辑所有字段
- 案件删除为**级联删除**：删除案件时，该案件下的所有文档、跟进记录、工作记录、时间线事件、待办事项将一并删除

## 6. 客户管理

客户管理与案件关联，方便跟踪同一客户下的多个案件。

### 功能

- **新增客户**：填写客户名称、类型（个人/企业）、联系方式、地址、身份证号/统一社会信用代码
- **编辑**：点击客户行修改信息
- **导出 Excel**：将客户列表导出为 `.xlsx` 文件，便于归档或汇报
- **关联案件**：客户信息自动填充到新建案件表单中

## 7. 传票管理

传票功能用于记录法院开庭传票的关键信息，进入案件详情页后点击「传票」标签页。

### 上传传票

1. 点击「上传传票」按钮
2. 选择传票文件（支持 PDF、图片格式）
3. 上传后出现在下方列表中

### 填写传票信息

1. 点击「手动填写」按钮，展开信息表单
2. 填写开庭相关信息：

| 字段 | 说明 |
|------|------|
| 开庭日期 | YYYY-MM-DD 格式 |
| 开庭时间 | 如 09:30 |
| 案号 | 传票上的案号 |
| 法院 | 传唤法院名称 |
| 开庭地点 | 如"第二审判庭" |
| 法官 | 承办法官 |
| 书记员 | 书记员姓名 |
| 联系电话 | 法院联系电话 |
| 案由 | 案件案由 |
| 传唤事由 | 开庭/谈话/调解/听证 |
| 被传唤人 | 被传唤当事人名称 |
| 被传唤人地址 | 送达地址 |

3. 点击「保存信息」

### 传票列表

每条传票支持：
- **预览** — 弹窗中查看传票文件内容
- **下载** — 下载传票原文件
- **编辑** — 点击传票条目修改信息
- **删除** — 删除该传票

> 💡 传票信息保存后会自动在案件时间线中添加开庭事件。

## 8. 文档管理

进入案件详情页后点击「文档」标签页。

### 功能

- **上传文档** — 支持 PDF、Word、Excel、PPT、图片、视频等格式
- **文件夹管理** — 创建文件夹对文档进行分类（如"证据材料""起诉材料""判决文书"）
- **列表/网格视图** — 切换文档显示方式
- **预览** — 支持以下格式在线预览：
  - PDF — 内嵌查看
  - 图片 — 大图预览
  - Word (.docx) — 转 HTML 渲染
  - Excel — 多 Sheet 分页浏览
  - 视频 — 内置播放器
- **下载** — 单文件下载或下载整个文件夹（打包为 ZIP）
- **删除** — 删除单个文档

### 支持的格式

图片：png, jpg, jpeg, gif, bmp, tiff, webp
文档：pdf, doc, docx, xls, xlsx, ppt, pptx, txt, rtf
压缩包：zip, rar, 7z
视频：mp4, avi, mov, mkv, wmv, flv, webm, m4v, 3gp

## 9. 时间线

案件时间线以可视化方式展示案件全生命周期中的关键事件。

### 添加事件

1. 进入案件详情页，点击「时间线」标签页
2. 点击「添加事件」
3. 填写日期、标题、描述、事件类型（立案/开庭/判决/调解/执行/其他）

### 事件类型

- 立案
- 开庭
- 判决
- 调解
- 执行
- 其他（证据交换、送达、上诉等）

### 自动联动

- 上传传票并填写开庭日期后，自动在时间线添加开庭事件
- 案件状态变更为「已结案」时可在时间线添加结案事件

## 10. 跟进记录

跟进记录用于记录与委托人、对方律师、法院的沟通情况。

1. 进入案件详情页，点击「跟进记录」标签页
2. 填写：日期、跟进内容、联系人、联系电话、跟进结果、下一步行动
3. 所有记录按日期排序，支持编辑和删除

---

## 11. 工作记录

工作记录用于记录律师在每个案件上的工时和工作内容。

1. 进入案件详情页，点击「工作记录」标签页
2. 填写：日期、工作内容、工时（小时）、工作类别（出庭/阅卷/起草/沟通/调研）
3. 仪表盘会汇总展示总工时

---

## 12. 待办事项

待办事项可以在左侧栏通用添加，也可以在案件详情页中添加。

### 优先级

- **紧急** — 红色标识
- **重要** — 黄色标识
- **普通** — 灰色标识

### 状态流转

待办 → 进行中 → 已完成

### 自动生成

- 提交传票信息后，自动生成「准备开庭材料」待办
- 截止日期临近的待办会在仪表盘高亮提示

## 13. 文书模板

文书模板用于快速生成标准化法律文书。

### 使用预设模板

系统内置以下模板：
- 民事起诉状
- 答辩状
- 律师函

### 自定义模板

1. 点击左侧栏「文书模板」
2. 点击「新建模板」
3. 使用 `{变量名}` 格式在模板中定义占位符
4. 使用时填入变量值，一键生成文书

### 上传模板

支持上传 Word (.docx) 和 PDF 格式的模板文件。

## 16. 常见问题

### Q: 数据存储在哪里？

所有数据存储在项目目录下 `data/cases.db`（SQLite 数据库文件）和 `data/uploads/`（上传文件目录）。建议定期备份这两个目录。

### Q: 如何备份数据？

```bash
# 备份整个数据目录
cp -r data/ ~/backup/case-manager-$(date +%Y%m%d)/
```

### Q: 如何迁移到另一台电脑？

将整个项目文件夹复制到新电脑，重新运行 `pip install -r requirements.txt` 和 `python app.py` 即可。

### Q: 为什么端口是 5066？

macOS 系统下 1024 以下的端口需要 root 权限，为避免权限问题，默认使用 5066 端口。如需修改，编辑 `app.py` 文件底部的 `port=5066` 参数。

### Q: 如何部署到服务器？

本项目设计为单机使用。如需多人协作，建议：
1. 使用 SQLite 的 WAL 模式 + 文件共享（如 NAS）
2. 或迁移到 PostgreSQL/MySQL 后端

### Q: 文件上传大小限制？

Flask 默认无文件大小限制。如需要，可在 `app.py` 中添加 `app.config['MAX_CONTENT_LENGTH']` 配置。
